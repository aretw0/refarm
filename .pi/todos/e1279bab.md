Backlog note from 2026-04-30: avoid waiting on Actions for every small slice; reproduce locally and monitor CI only for meaningful batches. Later investigate E2E bottlenecks without damaging the pinned/reusable/local-action architecture.

Evidence from recent runs:
- E2E job can take 3m31s to 13m07s.
- Setup dominates when Playwright install/cache misses occur (one run had Setup ~11m23s).
- Logs show `Cache not found` for `Linux-playwright-browsers-<sha>` and `Linux-turbo-<sha>`, followed by post-step warnings: `Input required and not supplied: path`.
- E2E currently runs `npx turbo run test:e2e --cache-dir=.turbo` without affected filter, putting 41 packages in scope.
- Playwright configs run `npm run build && npm run preview:test` inside webServer, causing rebuilds despite build artifacts.

Candidate slices:
1. Fix cache wrapper/nesting so Turbo and Playwright caches persist on miss.
2. Use `needs.changes.outputs.turbo_filter` for E2E with full fallback only when needed.
3. Let CI webServer preview prebuilt artifacts instead of rebuilding, while preserving local build+preview behavior.

Do this only when explicitly focusing on CI speed; otherwise continue product/extensibility work.

## 2026-05-01 execution update (in progress)

Pulled this backlog item and implemented a first acceleration slice:

1. **Cache strategy hardened in `.github/actions/setup/action.yml`**
   - Switched setup caches to direct pinned `actions/cache@27d5ce7f...` usage (Turbo, Playwright browsers, cargo-component) to avoid nested-wrapper post-step drift.
   - Turbo key changed from commit SHA to deterministic hash key:
     - `Linux-turbo-${hashFiles('package-lock.json','turbo.json')}`
   - Playwright key changed from commit SHA to deterministic hash key:
     - `Linux-playwright-browsers-${hashFiles('package-lock.json', playwright configs...)}`

2. **E2E scope now affected-first in `.github/workflows/test.yml`**
   - Added affected step:
     - `npx turbo run test:e2e --filter=${{ needs.changes.outputs.turbo_filter }} --cache-dir=.turbo`
   - Preserved full fallback when filter is empty.

3. **Avoid redundant webServer rebuild inside Playwright configs**
   - Added env-gated prebuilt mode (`REFARM_E2E_USE_PREBUILT=1`) for CI.
   - Added `preview:test:ci` scripts in both validation apps with safe fallback:
     - use existing `dist` when available
     - fallback to build+preview when artifact is absent
   - Updated both Playwright configs to use `preview:test:ci` in prebuilt mode.
