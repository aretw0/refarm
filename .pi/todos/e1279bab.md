Backlog note from 2026-04-30: avoid waiting on Actions for every small slice; reproduce locally and monitor CI only for meaningful batches. Later investigate E2E bottlenecks without damaging the pinned/reusable/local-action architecture.

Evidence from recent runs:
- E2E job can take 3m31s to 13m07s.
- Setup dominates when Playwright install/cache misses occur (one run had Setup ~11m23s).
- Logs show `Cache not found` for `Linux-playwright-browsers-<sha>` and `Linux-turbo-<sha>`, followed by post-step warnings: `Input required and not supplied: path`.
- E2E currently runs `npx turbo run test:e2e --cache-dir=.turbo` without affected filter, putting 41 packages in scope.
- Playwright configs run `npm run build && npm run preview:test` inside webServer, causing rebuilds despite build artifacts.

Candidate slices:
1. Fix cache wrapper/nesting so Turbo and Playwright caches persist on miss.
2. Use `needs.changes.outputs.turbo_filter` for E2E with full fallback only when needed.
3. Let CI webServer preview prebuilt artifacts instead of rebuilding, while preserving local build+preview behavior.

Do this only when explicitly focusing on CI speed; otherwise continue product/extensibility work.
