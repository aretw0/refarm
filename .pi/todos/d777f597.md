## Context

Assimilate DEC-031: Refarm should converge toward a unified `refarm` host analogous to Pi's single extensible process, while preserving Refarm's sovereign graph/plugin architecture.

## Target shape

- One canonical host/CLI initializes runtime, identity/storage/Tractor, plugin manifests, trust gates, telemetry, streams, and action routing.
- Renderers are interchangeable surfaces over the same contracts:
  - TUI renderer
  - Web renderer/Homestead server+browser
  - headless/agent mode
- `apps/*` remain product distributions/workbenches, not the only way to run Refarm.

## Next pragmatic slices

1. Define host boundary doc/ADR and package ownership.
2. Identify minimal TUI/Web renderer contract: slots, surfaces, actions, streams, status, telemetry.
3. Validate promoted DS/Homestead primitives with a second consumer before extracting more app/dev runtime.
4. Only then sketch `@refarm.dev/cli` or equivalent `refarm` command.
