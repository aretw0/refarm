# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish your code. You can
find the full documentation for it [in our repository](https://github.com/changesets/changesets).

We have a quick list of common questions to get you started engaging with this project in
[our documentation](https://github.com/changesets/changesets/blob/main/docs/common-questions.md).

---

## Quick Start for Refarm

### 1. Making Changes

When you make changes to any package, create a changeset:

```bash
npm run changeset
```

This will:

- Ask which packages changed
- Ask for a version bump type (major/minor/patch)
- Ask for a summary of changes
- Create a `.md` file in `.changeset/` folder

### 2. Versioning Packages

To bump versions based on changesets:

```bash
npm run changeset:version
```

This will:

- Update package.json versions
- Update CHANGELOG.md files
- Delete consumed changeset files

### 3. Publishing (Future)

When ready to publish to npm:

```bash
npm run build
npm run changeset:publish
```

## Integration with Turbo Repo

Changesets works seamlessly with Turbo:

- ✅ Understands workspace dependencies
- ✅ Updates internal dependencies automatically
- ✅ Respects package interdependencies

### Configuration

See [config.json](./config.json) for current settings:

- **access:** `public` (MIT license)
- **baseBranch:** `main`
- **updateInternalDependencies:** `patch` (bumps dependents when dependencies change)

## Best Practices

1. **Create changesets for every PR** — Don't merge without a changeset
2. **Be descriptive** — Changeset summaries become CHANGELOG entries
3. **One changeset per logical change** — Multiple related packages can share one changeset
4. **Version together** — Run `changeset:version` before releases, not per-commit
