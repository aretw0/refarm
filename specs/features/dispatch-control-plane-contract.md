# Feature: Dispatch Control Plane Contract (`@refarm.dev/dispatch-surface`)

**Status**: Accepted
**Owner**: Arthur Silva
**Date**: 2026-06-17

---

## Summary

This feature defines the stable transport/control contract shared by `@refarm.dev/dispatch-surface`, `apps/refarm`, and `apps/farmhand`.

It keeps CLI, sidecar runtime, and future channel adapters decoupled from runtime-specific assumptions while preserving:
- stable transport strings,
- deterministic route/path builders,
- explicit channel capability gating,
- canonical effort-envelope normalization.

It also includes the Rust/WIT bridge (`@refarm.dev/dispatch-surface-rs`) as an optional parity-checked implementation target.

---

## Contract Surface

### Canonical transports

Valid transports are:

- `file`
- `http`
- `channel:<name>`

`<name>` must be a non-empty string.

Parsing/validation requirement:

- accept exactly `file`, `http`, and `channel:<name>`
- reject all others with the stable message:

```text
Invalid task transport "<value>". Use: file, http, channel:<name>
```

### Channel effort payload normalization

A channel effort payload is valid when:

- `direction` is a non-empty string
- `tasks` is an array

Optional normalization behavior:

- `source` defaults to `channel:<name>`
- `context` includes normalized:
  - `channel`
  - `replyTo`
  - `traceIds`
- missing `submittedAt` is generated by the builder

### Path and encoding helpers

All channel route construction must go through shared builders:

- `buildChannelEffortsPath(baseUrl, channel)`
- `buildChannelEffortPath(baseUrl, channel, effortId, segment?)`
- `encodeChannel(channel)` / `decodeChannel(channel)`

### Control-surface capabilities

Channel control behavior is delegated through per-channel adapters:

- `submit`
- `query`
- `logs`
- `summary`
- `list`
- `retry`
- `cancel`

If a capability is disabled, transport/route paths for that action must be rejected deterministically (currently surfaced as 405 on the daemon side and deterministic adapter errors on the CLI side).

Adapters are exposed through:

- `resolveChannelControlSurfaceAdapter(channel)`
- `setChannelControlSurfaceAdapter(channel, adapter)`
- `removeChannelControlSurfaceAdapter(channel)`
- `getRegisteredChannelControlSurface(channel)`

Unknown channels must continue to resolve to a safe fallback adapter.

### Runtime host/daemon requirements

A compliant host/daemon must:

1. parse transport via `parseTaskTransport`
2. dispatch:
   - `file` → file transport adapter
   - `http` → HTTP worker transport
   - `channel:<name>` → channel-aware adapter path
3. build efforts via `buildChannelEffort`
4. preserve payload and normalizations above
5. enforce capability checks before each channel operation

## Conformance

- TS parity tests: `pnpm --filter @refarm.dev/dispatch-surface test:parity`
- Rust/WIT parity lane for dispatch-surface (when runtime artifacts are built)
- CLI regression coverage:
  - transport parsing and error messaging
  - channel URL/path builders for submit/query/logs/retry/cancel/summary/list
- Sidecar regression coverage:
  - capability-gated channel endpoints and unsupported operation behavior

---

## Non-goals

- This contract does **not** define plugin execution semantics.
- It does **not** define queueing/priority model internals beyond payload normalization.
