# XR Surface POC

**Status:** DRAFT - optional frontier packet
**Related:** `docs/CONVERGENCE_ROADMAP.md` planning frontier, `docs/UI_ARCHITECTURE.md`,
`specs/ADRs/ADR-070-wasm-surface-substrate.md`

## Context

XR should sit around Refarm as a surface, not inside the core runtime. Current WebXR guidance still
requires capability detection and secure contexts; MDN marks WebXR as limited availability and
experimental in production contexts. A-Frame 1.8.0 is a practical declarative layer over three.js
for quick WebXR scenes, while three.js remains the lower-level renderer path.

References:

- MDN WebXR Device API: https://developer.mozilla.org/en-US/docs/Web/API/WebXR_Device_API
- WebXR Device API draft: https://immersive-web.github.io/webxr/
- A-Frame 1.8.0 introduction: https://aframe.io/docs/1.8.0/introduction/
- three.js WebXR docs: https://threejs.org/docs/#api/en/renderers/webxr/WebXRManager

## Decision

Do not create a core XR package yet. The first XR work is a POC surface that consumes existing
Refarm contracts:

- `homestead` for surface/action envelope;
- `ds` tokens for non-XR controls and fallback UI;
- `dispatch-surface` for actions;
- optional A-Frame or three.js scene code isolated behind capability detection.

## Scope

- Build one static or dev-only XR scene that visualizes Refarm state or a generated vault map.
- Detect WebXR support and fall back to a 2D `homestead` view.
- Keep scene data as plain JSON generated by Refarm, not as hard-coded app state.
- Use A-Frame first for speed unless a lower-level three.js requirement appears.

## Non-goals

- No XR runtime dependency in core packages.
- No headset-only workflow.
- No WebXR-only admin feature.
- No multiuser or physics stack in the first POC.

## Proof

The POC is successful if a desktop browser sees the 2D fallback, an XR-capable browser can enter
the immersive scene, and both paths render the same Refarm-provided data.

## Gate

- Capability probe reports `supported`, `unsupported`, or `blocked`.
- 2D fallback is usable without XR APIs.
- Scene consumes data through a documented JSON envelope.
- No production package imports A-Frame or three.js unless the POC graduates.
