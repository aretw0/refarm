## Gate 2: Farmhand → Tractor Daemon Transition

The `tractor` Rust binary (graduated ADR-048, 2026-03-19) replaces the `apps/farmhand` Node.js daemon.

- [ ] `apps/farmhand` consumers confirmed to connect to `tractor` binary via `ws://localhost:42000`
- [ ] `BrowserSyncClient` interop tested end-to-end with production `.db` (not just `:memory:`)
- [ ] All 7 consumers in Consumer Map ([`packages/tractor/docs/ARCHITECTURE.md`](../packages/tractor/docs/ARCHITECTURE.md)) validated against `tractor` binary
- [ ] Schema migration tooling documented for users with existing `.db` files

## Gate 3: Homestead × Tractor Integration

### Gate 3a: Technical Primitives (prerequisite)

- [ ] `BrowserSyncClient` connects to tractor (`ws://localhost:42000`), receives initial snapshot
- [ ] Sync roundtrip: write via tractor → Loro delta → OPFS update in browser
- [ ] `installPlugin()` validates SHA-256, writes to OPFS, rejects tampered bundles

### Gate 3b: Reference Distro — `apps/me` (consolidated — this is the actual gate)

- [ ] `StudioShell` boots with Herald + Firefly active in real DOM slots (not placeholders)
- [ ] At least 1 sovereign content plugin loaded from OPFS with SHA-256 validated
- [ ] Offline-first confirmed: OPFS state persists, mutation replayed to tractor on reconnect

> Spec: [docs/gate3-homestead-tractor-spec.md](gate3-homestead-tractor-spec.md)
